# AutoML_course: Классификация состояния по данным ЭЭГ

## Описание задачи

**Задача:**  
На основе многоканального сигнала ЭЭГ определить вероятность двигательной активности по “эпохе” измерения (классы: “покой” или “движение рукой”) -> https://www.kaggle.com/competitions/neuroml2020eeg/overview.

**Данные:**  
- в Google Colab скачиваются из соответствующих скриптов в ноутбуках 
- 19 каналов, эпохи (срезы по времени), каждая эпоха - отдельный независимый пример  
- Целевая переменная отсутствует, вычисляется из поля `condition` (см. EDA)

## Репозиторий

- **part1_EDA:** исследовательский анализ данных, функции feature engineering
- **part2_modeling:** моделирование, подбор моделей & feature selection, финальные пайплайны
- **src/**: утилиты для фичей

## Основные шаги работы

### 1. EDA & Feature Engineering (`AutoML_course_part1_EDA.ipynb`)
- Численный и графический анализ исходных сигналов
- Верификация отсутствия утечек и корректного разбиения эпох
- Анализ распределений, корреляций, выбросов и пропусков
- Генерация доменно-специфичных признаков (например, p300-like фичи, относительная мощность β-ритма по каналам)
- Визуализации топографического распределения активности

### 2. Моделирование (`AutoML_course_part2_modeling.ipynb`)
- **Стратегия split**: только по эпохам, чтобы полностью исключить утечки между обучением и тестом
- **Базовые решения**:  
    - Несколько конфигов LightAutoML (LAMA):  
        - классика (ансамбль), fast(LGBM), ансамбль с расширенными моделями, вариант с feature selection  
    - Автоматический выбор наилучшего baseline  
- **Собственные пайплайны на sklearn/pandas**:  
    - LGBM + scaling  
    - CatBoost (с обработкой пропусков)  
    - Logistic Regression с гиперпараметрами (GridSearch)
- **Гипотезы и отчёты по метрикам ROC-AUC, F1**
- **Общий результат** — собственный пайплайн (логрегрессия+скалирование с подбором параметров с помощью Optuna) показал наилучший результат - линейные модели с осознанно-инженерными EEG-признаками работают не хуже автоэнсамблей в задаче среднего размера.
